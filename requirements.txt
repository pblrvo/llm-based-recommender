#    stages (see README's Setup section) -- the default PyPI wheel is

#      pip install torch==2.13.0 --index-url https://download.pytorch.org/whl/cu126
# 2. pip install -r requirements.txt        (everything below this point)
# 3. pip install unsloth==2026.7.2 --no-deps   (see note at the bottom)

# --- Why unsloth is installed separately with --no-deps ---
# unsloth==2026.7.2's own declared metadata pins torch<2.11.0, and
# unsloth_zoo (its own dependency) pins torch<2.13.0 -- both stale relative
# to what they actually run on. This project's own environment has run
# extensively on torch==2.13.0 + unsloth==2026.7.2 together (all of Stage 1/
# Stage 2 fine-tuning, every eval script) with no issues; `pip check` on
# that working environment still flags the same "conflict" satisfied here,
# confirming it's a packaging metadata lag, not a real incompatibility.
# Installing with --no-deps skips pip re-validating that stale pin -- safe
# here specifically because unsloth's actual runtime dependencies
# (transformers, trl, peft, bitsandbytes, datasets, etc.) are already
# pinned and installed above.
